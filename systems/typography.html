System / Typography

Typography is the first interface.

Redaction is the display and editorial layer. Geist is the reading and interface layer. Fontsource is the delivery layer: no Google Fonts import, no manual font upload.

Rule Set

Every text style has a strict job.

Use semantic HTML for structure, then apply font roles with discipline. Redaction creates identity. Geist carries information. Geist Mono carries technical metadata.

H1One page statement.

Use one real h1 per page. It names the page argument, not just the page category.

H2Section decision.

Use h2 to mark a new decision layer: system rule, example group, specimen, method or validation block.

PReading layer.

Paragraphs, body copy, labels, navigation and instructions use Geist. Redaction must not be used for continuous reading.

METATechnical layer.

Use Geist Mono for project codes, coordinates, versions, file names and compact system metadata.

Fontsource Pipeline

Fonts are dependencies, not uploaded assets.

The system imports Geist and Redaction through npm packages. The build process bundles only the selected weights and families.

Reading Layer

Geist carries the body.

Any text that asks the user to read, decide, navigate or act belongs to Geist. Redaction should never be the default bodycopy voice.

Always Geist. Default explanation layer.
Small

Never carry the main message.

Do / Don't

Do

Use Redaction 20 for XXL display.

Hero-scale titles need a cleaner Redaction state so identity stays expressive without becoming overly pixelated.

Don't

Use Redaction 100 for body copy.

Abstract states belong in motion, texture or transition moments. They are not final reading states.

Do

Use Geist for every reading layer.

Paragraphs, bodycopy, lead copy, navigation, labels, captions, buttons and instructions stay in Geist.

Don't

Ship font files as unmanaged uploads.

Fonts are installed through Fontsource and bundled through Vite so the system is reproducible.
