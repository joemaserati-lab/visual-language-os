System / Typography

Typography is the first interface.

Redaction gives identity. Geist gives clarity. The system separates expressive display states from readable information states.

Rule Set

Every text style has a job.

Use semantic HTML for structure, then apply visual scale through tokens. The page should still make sense if all colors are removed.

H1One page statement.

Use one real h1 per page. It names the page argument, not just the page category.

H2Section decision.

Use h2 to mark a new decision layer: system rule, example group, specimen, method or validation block.

H3Module title.

Use h3 inside cards, examples and small repeated modules. Do not use it as a smaller H2.

PExplanation layer.

Paragraphs use Geist because they must be read quickly. Redaction is reserved for high-impact display and transformation states.

Reference card

Readable identity first.

The title carries the tone. The paragraph carries the reasoning. The label carries metadata.

Good typography makes the reader understand the order of importance before they understand the details.

Do / Don't

Do

Use Redaction 35 for readable display.

Large editorial headlines, short statements and brand moments stay readable while still feeling distinct.

Don't

Use Redaction 100 for body copy.

Abstract states belong in motion, texture or transition moments. They are not final reading states.
