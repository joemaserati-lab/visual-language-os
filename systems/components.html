System / Components

Interactive states are part of the interface.

Every interactive component must communicate availability, intent and input feedback before motion or color become the only signal.

Component states

Document the full interaction loop.

Each state is shown as a visual frame with a descriptive label. Use these examples as acceptance criteria for navigation, calls to action, linked cards and interactive badges.

DefaultAvailable but calm.

The component is ready to use without competing with page hierarchy.

Hover / ActiveIntent and press feedback.

Hover confirms the target; active confirms that the input is currently being pressed.

Focus-visible / DisabledKeyboard clarity and unavailable actions.

Focus must be visible without mouse interaction. Disabled controls must look unavailable and should not be focusable when possible.
