System / Motion

Motion must reveal information.

Motion is not decoration. It is a sequencing layer that clarifies hierarchy, state, rhythm and attention.

Motion Laws

Movement needs a reason.

M-001Motion must have a job.

Reveal, connect, clarify, orient or confirm. If it does none of these, remove it.

M-002Readability must arrive early.

Scroll reveals complete before the element reaches the central focal zone.

M-003Transform state, not attention.

The user should understand what changed, not just notice that something moved.

Easing rules

Curves should feel calm, not decorative.

  • Use soft curves for reveals so content arrives without snapping.
  • Avoid decorative bounce; it adds personality without adding comprehension.
  • Use linear transitions only for technical transformations such as scrubbing, meters or synchronized geometry.

Trigger rules

Interaction decides when motion is allowed.

  • Hover is for local states.
  • Scroll is for editorial sequences.
  • Focus is for accessibility and keyboard orientation.
  • Click or tap is for interface confirmations.

Motion hierarchy

Scale motion to the size of the decision.

  • Micro motion confirms a single state change.
  • Component motion clarifies one reusable module.
  • Section motion sequences a group of related content.
  • Page motion orients the user during large context changes.

Forbidden motion

Some movement is never worth the cost.

  • No motion on body copy.
  • No delays on navigation.
  • No reveal that hides essential content.
  • No unnecessary decorative loops.
Do

Show the content first.

Do / Don't

Do

Use motion to expose sequence.

A section can move from abstract to readable when the transformation reinforces discovery.

Don't

Use motion to hide weak hierarchy.

If the page needs animation to become interesting, the static composition is not strong enough.
