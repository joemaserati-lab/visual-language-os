# Visual Language OS

Personal documentation system for typography, layout, motion, components and web design rules.

Current version: `v0.3 — Fontsource Typography Governance + Production Spec`

## Scope

This repository is a static documentation site for the Redaction + Geist visual language system.

The project is not a template. It is a decision system: principles, rules, anti-patterns, reference implementations and production guidelines.

## Font setup

Fonts are managed through Fontsource and bundled with Vite. The project no longer uses Google Fonts at runtime and no longer requires manual font uploads in `assets/fonts/`.

Installed families:

```txt
@fontsource-variable/geist
@fontsource-variable/geist-mono
@fontsource/redaction
@fontsource/redaction-10
@fontsource/redaction-20
@fontsource/redaction-35
@fontsource/redaction-50
@fontsource/redaction-70
@fontsource/redaction-100
```

Critical fonts are imported in `assets/js/app.js`. Redaction 50/70/100 are loaded conditionally only when a page contains progressive-recognition or pixel-state specimens.

## Core typographic decision

Geist is the default reading and interface layer. Paragraphs, body copy, lead copy, navigation, labels, captions, buttons and instructions use Geist.

Geist Mono is the metadata and technical layer: versions, labels, codes, token values, file names and system markers.

Redaction 10, 20 and 35 are the display/editorial layer. They create the visual signature of the portfolio.

Redaction 50, 70 and 100 are not static display fonts. They are transitional or experimental states.

Recommended motion sequence:

```txt
Redaction 100 -> Redaction 70 -> Redaction 50 -> Redaction 35
```

The transition must complete before the element reaches the user's visual focal zone.

## Production constraints

- Runtime Google Fonts: no.
- Manual unmanaged font uploads: no.
- Functional UI in Redaction: no.
- Body copy in Redaction: no.
- Inline prose links without affordance: no.
- Motion without reduced-motion fallback: no.
- Pixel states in global critical path without need: no.
