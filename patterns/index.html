Volume 3 / Pattern Library

Reusable editorial patterns.

A practical overview of page modules that preserve hierarchy, rhythm and recognition across the Design OS. Each pattern defines purpose, HTML structure, type, spacing, usage limits and a real visual sample.

Pattern 01

Hero statement

Scopo: declare the page thesis in one memorable statement before any supporting detail.

HTMLp.eyebrow + h1.display + p.hero-lead

Keep it inside main.content; use one real h1 only.

TypeMaximum contrast.

Redaction display for the statement, Geist lead for fast comprehension.

SpacingLet it breathe.

Use the default display margin, then separate the next module with .section.

Use when

A page needs a thesis.

Best for home, system chapters and major reference pages.

Avoid when

The task is transactional.

Do not slow down forms, utility pages or repeated index items.

Example

Motion must reveal information.

A large editorial claim followed by one concise explanatory line.

Pattern 02

Section intro

Scopo: introduce a new decision layer without competing with the page hero.

HTMLsection.section.grid

Put badge and h2 in a .span-5; explanatory rules or content in .span-7.

TypeEditorial but secondary.

Use h2 in Redaction and body copy in Geist with muted color.

SpacingOne section equals one decision.

Use .section top padding and border; avoid stacking unrelated intros inside the same section.

Use when

The user enters a new topic.

Ideal for rule sets, specimens, examples and methods.

Avoid when

The content is only a label.

Use an eyebrow or card title instead.

Rule Set

Every text style has a job.

Use semantic hierarchy first, then visual scale.

Pattern 03

Rule block

Scopo: turn a principle into an inspectable rule with a label, title and rationale.

HTMLdiv.rule

Use .rule-number, .rule-title and a short paragraph.

TypeDense, not loud.

Number labels are uppercase microcopy; titles use strong Geist for clear scanning.

SpacingBorder creates separation.

Let the built-in top border and padding separate rules; group multiple rules with .stack.

Use when

A decision must be repeatable.

Good for laws, states, accessibility limits and implementation notes.

Avoid when

You need storytelling.

Do not fragment narrative paragraphs into fake rules.

M-002Readability must arrive early.

Scroll reveals complete before the element reaches the central focal zone.

Pattern 04

Do/Don’t pair

Scopo: make a usage decision explicit by contrasting the recommended and risky choice.

HTMLdiv.do-dont with two .decision-cards

Apply .good and .bad to signal direction without relying only on wording.

TypeSymmetric hierarchy.

Both cards should use the same heading level and similar copy length.

SpacingTwo equal columns.

Keep the pair adjacent; on small screens the existing responsive rules stack them.

Use when

Behavior has a clear boundary.

Useful for motion, typography, contrast and interaction rules.

Avoid when

The answer is contextual.

If both choices may be valid, use a rule block instead.

Do

Reveal meaning.

Motion clarifies sequence.

Don’t

Hide controls.

Navigation must read instantly.

Pattern 05

Card grid

Scopo: present sibling options, volumes or modules with equal priority.

HTMLsection.section.grid + .card

Use .span-4 for three cards or .span-3 for four cards.

TypeSmall label, clear module title.

Use eyebrow metadata, h3 title and one concise paragraph.

SpacingEqual internal padding.

Let .card provide the internal rhythm; avoid adding custom margins between card children.

Use when

Items are peers.

Good for volumes, pattern links and feature groups.

Avoid when

Items need a sequence.

Use rule blocks or process steps for ordered decisions.

Pattern 06

Reference module

Scopo: show a finished page fragment so rules become tangible rather than abstract.

HTML.contrast-sample or .example-panel

Pair explanatory copy with a strong visual sample.

TypeCopy explains; art proves.

Use a readable body text block next to a large Redaction specimen or applied composition.

SpacingPanel owns its space.

Use a full-width bordered module; do not crowd it inside a dense text stack.

Use when

You need evidence.

Best after rules, before asking the user to adopt the pattern.

Avoid when

The example is generic.

A reference module must show a real system behavior.

Pattern 07

CTA closing block

Scopo: end a page with one decisive next step instead of a weak footer drift.

HTMLsection.section + .example-panel.dark

Use a badge, short heading, one paragraph and one text link or button-like anchor.

TypeOne action hierarchy.

The CTA title should dominate; supporting copy stays brief and readable.

SpacingClosing weight.

Place it near the end of content with section padding above; avoid multiple competing CTAs.

Use when

There is a logical next step.

Send users to a pattern, reference implementation or lab test.

Avoid when

No action is required.

Do not manufacture urgency on documentation pages.

Pattern 08

Progressive Recognition

Scopo: transform content from abstraction to readable meaning while preserving timing and comprehension.

HTML.recognition-word + data-progressive-recognition

Use it inside a demo, hero or transition panel; keep the final word short.

TypeRedaction state sequence.

Move from Redaction 100 through intermediate states to Redaction 35 before the focal zone.

SpacingLarge focal field.

Use enough vertical space for the transformation; the existing .demo min-height gives the word room.

Use when

Expression can precede reading.

Hero words, section transitions and large editorial statements.

Avoid when

Text is functional.

Never use it for navigation, buttons, labels or body copy.
